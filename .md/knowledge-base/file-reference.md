# Miko — File Reference

## Root Config

| File               | Purpose                                                                 |
| ------------------ | ------------------------------------------------------------------------ |
| `package.json`     | Project manifest — `dev`/`build`/`preview` scripts, React 18 + Vite 5 deps |
| `package-lock.json`| Locked dependency tree                                                    |
| `vite.config.js`   | Vite config — registers `@vitejs/plugin-react`, no other plugins/aliases  |
| `README.md`        | Project summary, setup instructions, structure overview, matching-logic explainer |
| `.claude/settings.local.json` | Local Claude Code permission settings for this project        |

## Source (`src/`)

| File                       | Exports / Role                                                                                                                                                                                                                                                                                          |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `src/main.jsx`             | React entry point — `createRoot`, renders `<App />` inside `<React.StrictMode>`, imports global `index.css`                                                                                                                                                                                              |
| `src/App.jsx`              | Root component — owns `activeTab` state (`ingredients` \| `recipes` \| `suggest`), renders the tab bar and header, calls `useStore()` once and passes the relevant slice of state/functions to whichever tab component is active                                                                        |
| `src/App.module.css`       | Layout, header, tab bar styles for `App.jsx`                                                                                                                                                                                                                                                             |
| `src/useStore.js`          | `useStore()` hook — the entire app's state and business logic. Holds `ingredients`, `recipes`, `nextId` (via `useState`); hard-codes `DEFAULT_INGREDIENTS` (checked ingredient list) and `DEFAULT_RECIPES` (16 seed recipes) as module-level constants. Exports `addIngredient`, `removeIngredient`, `toggleIngredient`, `addRecipe`, `removeRecipe`, `updateRecipeIngs`, `getMatch(recipe)`, `getSortedRecipes(skipSet)` |
| `src/Ingredients.jsx`      | Ingredients tab — text input + Add button (Enter key also submits) to add ingredients, in-stock checklist with checkbox-to-toggle and a per-row delete (×) button, "N of M in stock" counter, empty-state message                                                                                       |
| `src/Ingredients.module.css` | Styles for the ingredients list/rows/checkboxes/add row                                                                                                                                                                                                                                                |
| `src/Recipes.jsx`          | Recipes tab. Exports default `Recipes` component plus two file-local sub-components: `RecipeCard` (renders one recipe's match %, `MatchBar`, ingredient tag chips, and an "edit essential ingredients" toggle mode using ★ buttons) and `AddRecipeForm` (local form for name + ingredient list, each ingredient markable essential before saving). Splits recipes into makable and "Unmakable" sections via `getMatch(r).unmakable` |
| `src/Recipes.module.css`   | Styles for recipe cards, tag chips, essential-editing chip grid, add-recipe form                                                                                                                                                                                                                        |
| `src/Suggest.jsx`          | "Tonight's pick" tab — owns a `skipped` `Set` of recipe ids (local `useState`), calls `getSortedRecipes(skipped)` to rank remaining makable recipes, shows the top pick (name, reason text, `MatchBar`, ingredient tags, "Find recipe ↗" Google-search link, "Show another" skip button) plus up to 3 runner-up cards. Handles empty states (no recipes at all; no makable recipes with a "start over" reset link) |
| `src/Suggest.module.css`   | Styles for the top-pick card, runner-up cards, empty states                                                                                                                                                                                                                                              |
| `src/MatchBar.jsx`         | Shared, stateless. Exports `matchClass(pct)` (buckets a percentage into `'full'` \| `'partial'` \| `'low'`) and default `MatchBar({ pct })` (renders a filled progress bar sized to `pct`). Used by both `Recipes.jsx` and `Suggest.jsx`                                                                  |
| `src/MatchBar.module.css`  | Progress bar track/fill styles, including color variants per match class                                                                                                                                                                                                                                |
| `src/index.css`            | Global CSS custom properties for light/dark theming (no JS theme toggle — relies on `prefers-color-scheme` or similar in the stylesheet), base element resets                                                                                                                                            |
